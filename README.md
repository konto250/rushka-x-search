# ルーシュカ X検索ビルダー

ルーシュカ関連のX検索式をブラウザ上で組み立てる静的アプリです。

GitHubリポジトリ: https://github.com/konto250/rushka-x-search

基本検索式として、以下をベースに検索条件を追加できます。

```text
(#ルーシュカ OR ルーシュカ)
```

作成した検索式はそのままコピーでき、最新表示付きでX検索へ直接移動できます。

## ファイル構成

- `index.html`: 画面本体
- `styles.css`: 見た目のスタイル
- `script.js`: 検索式生成、コピー、Xリンク生成
- `vercel.json`: Vercel用の最小設定

## 使い方

1. `index.html` をブラウザで開きます。
2. 必要なオプションにチェックを入れます。
3. 必要に応じて除外ユーザーを入力します。
4. 生成された検索式を確認します。
5. `検索式をコピー` を押してコピーするか、`Xで開く（最新）` を押してX検索を開きます。

## オプション

### メディアのみ

以下を追加します。

```text
filter:media
```

### 引用リツイートを除外

以下を追加します。

```text
-filter:quote
```

### リツイートを除外

以下を追加します。

```text
-filter:retweet
```

### リプライを除外

以下を追加します。

```text
-filter:replies
```

### ユーザーを除外

チェックを入れると、除外したいユーザーIDをスペース区切りで入力できます。

入力例:

```text
user1 user2
```

生成される検索条件:

```text
-from:user1 -from:user2
```

`@` を付けて入力しても、自動で取り除いて検索式に反映します。

## 生成されるXリンク

X検索リンクには、常に最新表示用の `f=live` パラメーターが付きます。

例:

```text
https://x.com/search?q=...&f=live
```

## ローカルで開く方法

このアプリは静的ファイルだけで動作します。ビルドは不要です。

以下のどちらかで利用できます。

1. `index.html` をブラウザで直接開く
2. 任意の静的サーバーで配信する

## Vercelへデプロイ

このプロジェクトフォルダで以下を実行します。

```powershell
npx vercel login
npx vercel --prod
```

初回はVercelの案内に従ってプロジェクトを関連付けてください。

## 補足

- コピー機能は `navigator.clipboard` を優先して使います。
- 利用環境によっては、フォールバックとして手動コピー用の選択処理が使われます。
